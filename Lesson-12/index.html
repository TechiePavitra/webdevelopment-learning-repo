HTML Entities — Examples

What are HTML entities?

HTML entities are special codes used to represent characters that either have special meaning in HTML (like < or >) or characters that are not present on your keyboard. Use named entities (like &amp;) or numeric references (decimal or hexadecimal) to display those characters.

Common named entities

  • &amp; → &
  • &lt; → <
  • &gt; → >
  • &quot; → " (double quote)
  • &apos; → ' (apostrophe; supported in HTML5)
  • &nbsp; → non-breaking space
  • &copy; → © (copyright)
  • &reg; → ® (registered)
  • &euro; → € (euro sign)

Numeric references

Numeric references use the character's Unicode code point. Decimal format: &#169;. Hex format: &#xA9;.

Examples:

  • &#169; or &#xA9; → ©
  • &#241; or &#xF1; → ñ

Why use entities?

Use entities to:

  • Display characters that would otherwise be interpreted as HTML (for example: &lt; for <).
  • Include special symbols (©, ®, €, →) safely in your markup.
  • Insert non-breaking spaces (&nbsp;) to prevent unwanted line breaks.

In an attribute: <a title="Tom &amp; Jerry">hover me</a> renders as: hover me

Notes and tips

  • Named entities are easier to read, but not all characters have named entities — numeric references always work.
  • When showing HTML code on a page (for teaching), escape the angle brackets with entities (for example: &lt;div&gt;).
  • HTML5 supports many named entities; prefer the standard ones (&amp;, &lt;, &gt;, &quot;, &nbsp;).
